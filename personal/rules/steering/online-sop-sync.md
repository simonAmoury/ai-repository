# 外部改动即同步上线 SOP

改动涉及**外部系统/平台对接内容**（Apollo 配置项、飞书卡片模板、第三方接口约定、DDL，等任何超出本仓库代码范围、需人工在外部平台操作才能生效的内容）时：

- **改动发生的当次**就同步记录到项目的上线 SOP 文档（新增小节或追加条目），不要攒到功能全部完成后再一次性回忆整理。
- 每次涉及外部改动的代码提交，视为同时完成了对应 SOP 条目的编写。
- 若项目还没有上线 SOP 文档，则新建一份；编写 / 补充时套用 `write-online-sop` skill 规定的结构与条目格式（Apollo 配置须含 Key/命名空间/处理/不处理的后果，可选项显式标注）。

改动涉及**既有** Apollo 配置项（修改某 key 的用途、格式、取值范围，或该配置被多处读取）时，主动提醒检查该配置在各环境（测试/线上）的当前值是否仍然适用，不要假设无需变动。
